Performance testing notes

Load testing

Testing system's performance under normal and expected load

Spike testing

testing system's performance under sudden and extreme load increases

Stress testing

type of performance testing where the system is pushed to its limit to determine its breaking point

Important performance metrics

http_req_duration

This is the time taken to complete the http requests

avg=143.92ms min=17.09ms med=240.75ms max=616.48ms p(90)=262.72ms p(95)=273.46ms

Here is an interpretation of the result above:

  • avg: average time for the requests was 143.92ms
  • min: fastest request took 17.09ms
  • med: half of the requests took less than 240.75ms
  • max: slowest request took 616.48ms
  • p(90): 90% of requests took 262.72ms or less
  • p(95): 95% of requests took 273.46ms or less

Why p(95) is more useful than avg

avg can get skewed by outliers. One slow request can ruin the entire result

p(95) is more reliable because it shows us how consistent the performance is.

p(95) shows us what most users experience

http_req_failed

This is the percentage of failed requests

http_req_failed................: 0.00% 0 out of 402

This shows that 0% of requests failed

http_req_waiting

Time spent waiting for the server to respond

avg=295ms min=12.16ms med=239.26ms max=2.8s p(90)=996.28ms p(95)=1.27s

Here is an interpretation of the result above:

  • avg: average server response was 295ms
  • min: fastest server response time was 12.16ms
  • med: half of the requests had a TTFB(Time to First Byte) less than 239.26ms
  • max: slowest server response 2.8s
  • p(90): 90% of requests had a TTFB OF 996.28ms or less
  • p(95): 95% of requests had a TTFB OF 1.27s or less

TTFB- time between sending a request to a server and receiving the first byte of the response
